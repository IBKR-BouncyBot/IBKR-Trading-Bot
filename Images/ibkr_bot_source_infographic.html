1
ATR WARMUP + ANCHOR

Collect RTH data, then watch for the initial drop

Only newly consumed ticker events can warm ATR, set the anchor, or advance the strategy.

  • Observed prices are bucketed into 60-second OHLC bars; ATR period 14 needs 15 distinct bar buckets.
  • During default warmup, Stage 1 has no armed drop trigger; the manual drop percentage is ignored.
  • When ATR becomes ready, that price becomes a fresh anchor; a later event must make the drop.
  • Before entry, any new high ratchets the anchor upward.
Stage-1 trigger
drop trigger = anchor × (1 − drop% / 100)
Entry becomes eligible when selected price ≤ trigger.
Adaptive defaults: drop 1.50× ATR%, BUY trail 0.75×, minimum profit 1.00×, SELL trail 1.00×. Clamp: 0.10%–20.00%.
Manual drop fallback 2.00%RTH-only samples
2
ENTRY ORDER

Submit the BUY trail and size whole shares

  • At the drop condition, calculate a projected BUY stop above the current price.
  • Default budget is $10,000 + positive completed app net P/L for the ticker because reinvestment is ON.
  • Positive trail → native IBKR BUY TRAIL. A 0.00% trail → immediate market BUY.
  • After any positive fill, record the actual average fill, cancel the unfilled remainder, and manage only the filled quantity.
Entry math
BUY stop = price × (1 + BUY trail%)
qty = floor(budget / sizing price)
Slippage buffer is OFF by default, so sizing price equals the projected stop.
Order: GTC, outsideRth=False, whole shares, app-owned IBKRBOT|… reference.
Manual BUY fallback 1.00%Partial fill accepted
3
POSITION OPEN

Wait until the final SELL can protect minimum gross profit

The profit reference is the actual average BUY fill — not the Stage-1 anchor.

  • Default minimum profit is ATR-adapted at 1.00× ATR%; manual fallback is 3.00%.
  • Default final SELL trail is ATR-adapted at 1.00× ATR%; manual fallback is 1.00%.
  • The final exit is not submitted until the selected price reaches the required level and SELL checks permit submission.
  • Protective SELL is OFF by default. If enabled, it is placed after a positive BUY fill.
Profit activation
minimum SELL stop = avg BUY × (1 + min profit%)
required price = minimum stop / (1 − SELL trail%)
Commissions and actual market-order slippage are not guaranteed by this threshold.
Optional protective trail default: 3.00%; protective ATR adaptation is OFF.
Slippage planning OFFNo fixed profit guarantee
4
PROFIT-PROTECTING EXIT

Cancel/replace safely, then exit through IBKR

  • Positive SELL trail → native IBKR SELL TRAIL for remaining app-owned shares.
  • A 0.00% SELL trail → market SELL once the minimum-profit threshold is reached.
  • If a protective SELL is working, request cancellation and wait until it is confirmed nonworking before final SELL submission.
  • After acceptance, IBKR controls the live trailing stop; the app polls and records broker state.
Order construction
SELL TRAIL • remaining whole shares
GTC • outsideRth=False • IBKRBOT|…
Initial stop is rounded down to minimum tick and may not be below the required profit floor.
Market-style execution warning: gaps, latency, liquidity and broker behavior can fill below the projected stop or threshold.
5
RECORD + REPEAT

Persist the completed cycle and optionally start the next one

  • SQLite records orders, executions, commissions, gross/net P/L, UTC timestamps, audit and recovery events.
  • Auto-repeat ON; another Stage-1 cycle starts unless stop-after-current-cycle or an enabled completed-cycle cap stops it.
  • Reinvest profits ON; only positive completed app net P/L is added to the next base budget.
Recovery: reconcile persisted state, open app-owned orders, recent executions, contract/account facts and unsold local fill quantity.

Verified new-configuration defaults

ON by default

  • ATR adaptive percentages
  • Adapt minimum profit with ATR
  • Block BUY until ATR ready
  • Block delayed data in live mode
  • IBKR what-if preflight
  • Stale-data guard
  • Session-timing guard
  • RTH-only order submission
  • Auto-repeat

OFF by default

  • Protective SELL
  • Protective SELL ATR adaptation
  • Slippage planning buffer
  • Hard risk-limit master
  • Recent-volatility filter

Key numeric defaults

  • Investment: $10,000; exchange SMART; USD stock; GTC
  • ATR: 14 periods, 60-second bars, 15 buckets to ready
  • Freshness: price 3 s, bid/ask 3 s, RTH status 60 s
  • Session: first 5 min / last 15 min / cancel BUY 5 min
  • Manual fallbacks: drop 2% / BUY 1% / profit 3% / SELL 1%
  • Stored inactive values: protective 3%, slippage 0.5%, hard spread 1%
